# Visual Explainer Handoff

You are collaborating with an implementation steward. Provide a committed
visual direction for a standalone educational HTML explainer; do not edit the
codebase.

Constraints:
- Ground the visual system in the explainer topic.
- Choose a distinct palette, typography pairing, layout language, and diagram
  style rather than defaulting to a generic documentation page.
- Account for light and dark mode from the start.
- Prefer concrete composition and visual-device guidance over abstract taste
  words.
- Keep the handoff concise enough to implement directly.

Return:
1. Design concept and visual metaphor.
2. Palette and typography direction.
3. Page composition and information hierarchy.
4. Recommended diagrams, worked examples, or interactive moments.
5. Light/dark-mode considerations.
6. Risks the implementation steward should guard against.
